# 第5节：申请人联系方式 (Applicant Contact Information) + 社交媒体

申请人联系方式

## 子标题1：申请人家庭住址

### 1. 申请人家庭住址 / Home Address
- **字段类型**: 地址表单组
- **布局**: 页面顶部
- **子字段**:
  - **国家** (ID: `data.home_addr.country`) - 下拉选择框，必填，约280个选项，中文显示
  - **州/省/地区** (ID: `data.home_addr.state`) - 文本输入框，必填
  - **市** (ID: `data.home_addr.city`) - 文本输入框，必填
  - **街道地址** (ID: `data.home_addr.street_addr1`) - 文本输入框，必填
  - **邮编/邮政编码** (ID: `data.home_addr.zip_code`) - 文本输入框，必填

## 子标题2：联系信息

### 2. 最常用的电话号码 / Primary Phone Number
- **字段ID**: `data.phone_primary`
- **字段类型**: 文本输入框 (text input)
- **必填**: 是 (标记 *)
- **布局**: 第2个字段
- **验证**: 只接受数字（0-9），不接受连字符、空格等特殊字符

### 3. 备用电话号码 / Secondary Phone Number
- **字段ID**: `data.phone_secondary`
- **字段类型**: 文本输入框 (text input)
- **必填**: 否
- **布局**: 第3个字段
- **说明**: 如果您没有辅助电话号码，请保留空白。
- **验证**: 只接受数字（0-9）

### 4. 工作电话号码 / Work Phone Number
- **字段ID**: `data.phone_work`
- **字段类型**: 文本输入框 (text input)
- **必填**: 否
- **布局**: 第4个字段
- **说明**: 如果没有工作电话号码，请保留空白。
- **验证**: 只接受数字（0-9）

### 5. 电子邮件 / Email Address
- **字段类型**: 电子邮件表单组（2个输入框）
- **必填**: 是 (标记 *)
- **布局**: 第5-6个字段
- **子字段**:
  - **输入电子邮件** (ID: `data.email`) - 文本输入框，必填
  - **确认邮件** (ID: `data.email_confirm`) - 文本输入框，必填，必须与第一个邮箱一致
- **说明**: 请输入有效的电子邮件地址。它将用于就您的应用程序与您联系。

## 子标题3：其他联系方式

### 6. 在过去的五年中，您是否使用过其他电话号码？
- **字段类型**: 单选按钮组 (radio group)
- **必填**: 是 (标记 *)
- **布局**: 第7个字段
- **选项值**:
  - `true` - 有
  - `false` - 没有
- **衍生字段**: 当选择"有"(true)时，显示其他电话号码输入列表：
  - 其他电话号码 - 文本输入框，必填
  - "加上另一个"按钮 - 可添加多个电话号码

### 7. 在过去五年中，您是否使用过其他电子邮件地址？
- **字段类型**: 单选按钮组 (radio group)
- **必填**: 是 (标记 *)
- **布局**: 第8个字段
- **选项值**:
  - `true` - 有
  - `false` - 没有
- **衍生字段**: 当选择"有"(true)时，显示其他电子邮件输入列表：
  - 其他电子邮件地址 - 文本输入框，必填
  - "加上另一个"按钮 - 可添加多个电子邮件

### 8. 邮寄地址 / Mailing Address
- **字段ID**: `data.mailing_address_different`
- **字段类型**: 复选框 (checkbox)
- **必填**: 否
- **布局**: 第9个字段
- **标签**: 邮寄地址与申请人地址不同
- **衍生字段**: 当勾选时，显示完整邮寄地址表单：
  - 国家 - 下拉选择框，必填
  - 州/省/地区 - 文本输入框，必填
  - 市 - 文本输入框，必填
  - 街道地址 - 文本输入框，必填
  - 邮编/邮政编码 - 文本输入框，必填

### 10. 社交媒体标识符 / Social Media Identifier
- **字段ID**: `data.social_media_info[0].username`
- **字段类型**: 文本输入框 (text input)
- **必填**: 是 (标记 *)
- **布局**: 第11个字段
- **说明**: 填写您在该平台的用户名（或账号名），请勿填写密码

### 11. 加上另一个 / Add Another
- **字段类型**: 按钮 (button)
- **布局**: 第12个字段

- **功能**: 添加另一个社交媒体平台条目
- **衍生字段**: 点击后动态添加新的平台+用户名表单组

### 12. 您是否希望提供有关您在过去五年中用于创建或共享内容（照片，视频，状态更新等）的任何其他网站或应用程序上的状态的信息？

  - `true` - 有

- **布局**: 第11个字段
- **条件显示**: 当平台选择"无"时，此字段隐藏；选择任何其他平台时显示
- **说明**: 填写您在该平台的用户名（或账号名），请勿填写密码

## 字段交互逻辑

### 条件字段显示规则

1. **其他电话号码**:
   - 触发条件: "在过去的五年中，您是否使用过其他电话号码" 选择 "有"(true)
   - 衍生字段: 可能显示其他电话号码输入框

2. **其他邮箱**:
   - 触发条件: "在过去五年中，您是否使用过其他电子邮件地址" 选择 "有"(true)
   - 衍生字段: 可能显示其他邮箱输入框

3. **邮寄地址**:
   - 触发条件: 勾选"邮寄地址与申请人地址不同"
   - 衍生字段: 可能显示单独的邮寄地址表单

4. **社交媒体**:
   - 点击"加上另一个"按钮后，动态添加新的社交媒体平台+用户名表单组
   - 支持添加多个社交媒体条目

---

## 技术细节

- Ant Design Select 组件用于下拉框
- 复选框使用 `ant-checkbox` 组件
- 社交媒体表单支持动态添加（类似数组结构）
- 表单验证在点击"下一个"时触发
- 错误信息以内联方式显示在字段下方
