## 字段交互逻辑

### 条件字段显示规则

1. **所有32个问题统一交互模式**:
   - 触发条件: 任意问题选择 "有"(yes)
   - 衍生字段: 在该问题下方立即显示"请详细说明"多行文本框 (textarea)，必填
   - 切换为"没有"时，文本框隐藏，已填写内容保留在 securityAnswers 中

2. **数据结构**:
   - 所有答案存储在 `securityAnswers: Record<number, { answer: string; explain: string }>`
   - key 为问题编号 (1-32)
   - 首次访问某问题时通过 `ensureAnswer(num)` 初始化

3. **Part 分组显示**:
   - 每个 Part 开头显示分组标题（h3 标签）
   - Part 之间无额外交互差异，仅视觉分组

---

## 技术细节

- 全部为单选按钮组（有/没有），字段值存储为字符串 "yes" 或 "no"
- 共32个radio group，分为5个Part（3 + 7 + 12 + 5 + 5）
- 每个问题通过 i18n key 引用（usVisa.security.q1 ~ q32）
- Part 标题通过 i18n key 引用（usVisa.security.part1 ~ part5）
- securityAnswers 使用 Record 类型，按需初始化
- 选择"有"后显示 Explain textarea，textarea 为必填
- 表单验证在点击"下一个"时触发，所有问题必须回答
